9.6 发布 LaTeX 文档

这个发布风格可以生成 LaTeX 或者 PDF 文档。

如果你想要发布 PDF 文档，你需要有一个安装好的 TeX。对于 Debian 和 Ubuntu 可以安装“tetex-bin”和“tetex-extra”包。TeX 字体也是必须的。

提供的风格

latex
发布一个 LaTeX 文档。


pdf
使用一个外部的 LaTeX 文档转换工具，发布一个 PDF 文档。


latexcjk
发布一个 CJK（中文）编码的 LaTeX 文档。


pdfcjk
使用一个外部的 LaTeX 文档转换工具，发布一个 CJK（中文）编码的 PDF 文档。

提供的选项

muse-latex-extension
发布 LaTeX 文件默认的扩展名。
muse-latex-pdf-extension
发布 LaTeX 文件到 PDF 默认的扩展名。
muse-latex-header
发布 LaTeX 文件使用的页眉。

这可以是文本或者一个文件名。

muse-latex-footer
发布 LaTeX 文件使用的页脚。

这可以是文本或者一个文件名。

muse-latexcjk-header
发布 LaTeX 文件（CJK）使用的页眉。

这可以是文本或者一个文件名。

muse-latexcjk-footer
发布 LaTeX 文件（CJK）使用的页脚。

这可以是文本或者一个文件名。

muse-latex-markup-regexps
在一个 Muse 页面中用来识别区域的标记正则表达式表。

关于这个列表构成的更多信息，参阅 muse-publish-markup-regexps。

muse-latex-markup-functions
为该风格文本定制函数的风格样式列表。

关于这个列表构成的更多信息，参阅 muse-publish-markup-functions。

muse-latex-markup-strings
标记文本使用的字符串。

这些字符串涵盖最基本的标记类，它们在各种风格之间的操作差别很小。

muse-latexcjk-encoding-map
映射 emacs 编码系统到合适 CJK 编码的一个列表。使用编码系统的基名（例如， 没有 -unix）。
muse-latexcjk-encoding-default
在发布文件中使用默认的 Emacs buffer 编码。

如果没有发现特殊字符，该选项将被使用。

muse-latex-markup-specials-example
必须被特殊表示的字符列表，用于 example 区域。

在 <example> 区域的缺省解释下，没有需要被转义的特别字符。

muse-latex-markup-specials-document
必须被特殊表示的字符列表，用于整个文档中（除去已经被转义的区域）。
muse-latex-markup-specials-literal
必须被特殊表示的字符列表，用于 =monospaced text= 和 <code> 区域。
muse-latex-markup-specials-url
必须被特殊表示的字符列表，用于 URL。
muse-latex-permit-contents-tag
如果该选项为空，省略 <contents> 标签；否则，插入目录。

大多数时候，最好在第一页有一个目录，紧跟其后一个新页。为了让这个设置在 发布到 HTML 和 LaTeX 文档时都能够工作，我们需要省略 <contents> 标签。

如果你不赞成这样，那么设置这个选项非空，你就会得到你所期望的了。
